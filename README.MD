# Design Patterns Collection

This repository serves as a centralized collection of classic software design patterns implemented in C#. Each pattern is structured in its own directory, containing a detailed `README.md` with problem-solution analysis and UML structure.

## Patterns Index

| Pattern | Description | Directory |
| :--- | :--- | :--- |
| **Abstract Factory** | Creates families of related objects. | [View](./AbstractFactoryPatternSolution/) |
| **Builder** | Constructs complex objects step-by-step. | [View](./BuilderPatternSolution/) |
| **Factory Method** | Defines an interface for object creation. | [View](./FactorymethodPatternSolution/) |
| **Prototype** | Creates new objects by cloning instances. | [View](./PrototypePatternSolution/) |
| **Singleton** | Ensures a class has a single instance. | [View](./SingletonSolution/) |

# مجموعه الگوهای طراحی (Design Patterns)

این مخزن به عنوان مجموعه‌ای متمرکز از الگوهای طراحی کلاسیک نرم‌افزار که با زبان C# پیاده‌سازی شده‌اند، عمل می‌کند. هر الگو در پوشه مخصوص به خود ساختاردهی شده است و شامل یک فایل `README.md` دقیق با تحلیل مسئله-راه‌حل و ساختار UML می‌باشد.

## فهرست الگوها

| الگو | توضیحات | پوشه |
| :--- | :--- | :--- |
| **Abstract Factory** | ایجاد خانواده‌هایی از اشیاء مرتبط. | [مشاهده](./AbstractFactoryPatternSolution/) |
| **Builder** | ساخت مرحله به مرحله اشیاء پیچیده. | [مشاهده](./BuilderPatternSolution/) |
| **Factory Method** | تعریف رابطی برای ایجاد اشیاء. | [مشاهده](./FactorymethodPatternSolution/) |
| **Prototype** | ایجاد اشیاء جدید از طریق کپی کردن نمونه‌ها. | [مشاهده](./PrototypePatternSolution/) |
| **Singleton** | تضمین اینکه یک کلاس فقط یک نمونه دارد. | [مشاهده](./SingletonSolution/) |

## راهنمای مستندات

هر زیرپوشه شامل یک فایل `README.md` جامع است که ویژگی‌های زیر را دارد:

1.  **مرور کلی**: تعریف و هدف الگو.
2.  **پیاده‌سازی**: بررسی دقیق کد C# و مثال‌های کاربردی.
3.  **ساختار UML**: نمودارهای کلاس استاندارد که مکانیسم الگو را نشان می‌دهند.
4.  **ساختار UML پیاده‌سازی پروژه**: نمودار ساختار کد اختصاصی در این پروژه.
