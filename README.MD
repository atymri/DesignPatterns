# Design Patterns Collection

This repository is a collection of classic design patterns implemented in C#. My goal is to make these patterns easy to understand by showing not just the theory, but how they actually look in code.

## Why this repository?
*   **Direct Mapping**: I've included diagrams for the standard theory (GoF) and separate diagrams for my actual C# implementation, so you can see exactly how the code connects to the concept.
*   **Dual-Language**: Documentation is available in both English and Persian for easier learning.
*   **Keep it Simple**: No fluff, just clean implementations and clear explanations to help you get the idea and use it in your projects.

## Patterns Index

| Pattern | Description | Directory |
| :--- | :--- | :--- |
| **Abstract Factory** | Creates families of related objects. | [View](./AbstractFactoryPatternSolution/) |
| **Builder** | Constructs complex objects step-by-step. | [View](./BuilderPatternSolution/) |
| **Factory Method** | Defines an interface for object creation. | [View](./FactorymethodPatternSolution/) |
| **Prototype** | Creates new objects by cloning instances. | [View](./PrototypePatternSolution/) |
| **Proxy** | Provides a surrogate or placeholder. | [View](./ProxyPatternSolution/) |
| **Singleton** | Ensures a class has a single instance. | [View](./SingletonSolution/) |

# مجموعه الگوهای طراحی (Design Patterns)

اینجا مجموعه‌ای از الگوهای طراحی کلاسیک رو با C# پیاده کردم. هدفم این بوده که یادگیری این الگوها فقط در حد تئوری باقی نمونه و دقیقاً بفهمیم توی کد چه اتفاقی می‌افته.

## چرا این مخزن؟
*   **ارتباط مستقیم با کد**: برای هر الگو هم نمودار تئوری (GoF) رو گذاشتم و هم نمودار مخصوصِ کدی که همین‌جا پیاده کردم؛ اینطوری دقیقاً می‌فهمید تئوری چطور تبدیل به کد شده.
*   **دو زبانه**: کل مستندات هم به انگلیسی هست و هم به فارسی، تا هر کدوم رو راحت‌تر بودید بخونید.
*   **ساده و کاربردی**: خبری از توضیحات اضافی و کتابی نیست. سعی کردم پیاده‌سازی‌ها تمیز باشن و توضیحات هم مستقیم برن سر اصل مطلب.
